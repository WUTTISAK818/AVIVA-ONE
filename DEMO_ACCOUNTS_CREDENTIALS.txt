   1. Login as demo.finance@alisa.com

   3. See approval queue

   5. See approval history

   1. Login as demo.sales@alisa.com

   3. Expected: Should get "Access Denied" or redirect to allowed page

   1. Open DevTools (F12)
   2. Go to Network tab
   3. Refresh page (Cmd+R)
   4. Wait for all requests to complete
   5. Look at timing
   6. Expected: Total load time < 2000ms

================================================================================
ISSUES TO LOOK FOR
================================================================================

Critical Issues (MUST FIX before go-live):
   ❌ Blank screen / 500 error
   ❌ Login not working
   ❌ Data not loading
   ❌ RBAC not enforced (e.g., Sales can see Finance)
   ❌ Links between modules broken (CRM ↔ Construction)
   ❌ Database not responsive

Important Issues (SHOULD FIX before go-live):
   ⚠️  Page takes > 3 seconds to load
   ⚠️  Form validation missing
   ⚠️  Missing required features
   ⚠️  Typos in Thai/English text
   ⚠️  Buttons not clickable
   ⚠️  No error messages shown

Minor Issues (CAN FIX after go-live):
   ℹ️  UI alignment/spacing
   ℹ️  Color scheme
   ℹ️  Confirmation messages
   ℹ️  Tooltip text
   ℹ️  Menu order

================================================================================
SUPPORT & TROUBLESHOOTING
================================================================================

Issue: Cannot login
   → Check password is EXACT: Demo@[Role]123 (Case sensitive!)
   → Check email is correct
   → Check user exists in Supabase

Issue: Page takes too long to load
   → Check network connection (DevTools → Network tab)
   → Check browser console for errors (F12)
   → Try clearing cache (Cmd+Shift+Delete)

Issue: RBAC not working (e.g., Sales can see Finance)
   → Check user role in Supabase
   → Check role is assigned correctly
   → Browser cache might be old - clear and retry

Issue: Data not displaying
   → Check Supabase connection
   → Check data exists in database
   → Check console for API errors
   → Try refresh page

Issue: Cannot edit something that should be editable
   → Check your role/permission
   → Check if it's in read-only mode
   → Check console for permission denied error
