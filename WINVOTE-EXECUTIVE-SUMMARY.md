# WinVote — บทสรุปผู้บริหาร (Executive Summary)
### โครงสร้างแอป + การใช้งานจริง

> เอกสารระดับผู้บริหาร — เน้นภาพรวม คุณค่า และการใช้งานจริง (ไม่ลงรายละเอียดโค้ด)
> อ้างอิงรายละเอียดเชิงลึก: `WINVOTE-VERIFICATION-DESIGN.md` · `WINVOTE-EXPERT-AUDIT.md`

---

## 1. WinVote คืออะไร
ระบบบริหาร **เครือข่ายฐานเสียงเลือกตั้งท้องถิ่น** บนมือถือ — เปลี่ยนการ "หาเสียงด้วยความรู้สึก"
ให้เป็น **ข้อมูลที่ตรวจสอบได้ + พยากรณ์โอกาสชนะได้** ตั้งแต่ระดับหัวคะแนนรายคน ไปจนถึงภาพรวมทั้งเทศบาล

---

## 2. โครงสร้างองค์กรในระบบ (ใครทำอะไร)
```
ผู้บริหาร (exec)               เห็นทุกเขต · ตั้งเป้า · วิเคราะห์ · เตรียมนำเสนอ
   └─ สตาฟประจำเขต (6+ คน/เขต)   ดูแลเฉพาะเขตตัวเอง · ประสานหัวหน้าทีม
        └─ หัวหน้าทีม            คุมยอดรวมลูกทีมในทีม · ปิดยอด · เคลียร์ปัญหา
             └─ ลูกทีม           รับผิดชอบฐานเสียง 50 คน · เก็บ+ยืนยันตัวตน
```
**สิทธิ์เห็นข้อมูลแยกตามชั้น** — ลูกทีมเห็นของตัวเอง, หัวหน้าทีมเห็นทั้งทีม, สตาฟเห็นทั้งเขต, ผู้บริหารเห็นทั้งหมด

---

## 3. ระบบแบ่งเป็น 3 ส่วน
| Part | ทำอะไร | ใครใช้ |
|---|---|---|
| **1. เก็บ + ยืนยันฐานเสียง** | เก็บผู้มีสิทธิ์ 50 คน/ลูกทีม + พิสูจน์ว่าจริง | ลูกทีม |
| **2. รวมยอด + กระทบยอด** | รวมยอดทีม/เขต + ตรวจสอบความถูกต้อง | หัวหน้าทีม/สตาฟ |
| **3. วิเคราะห์โอกาสชนะ** | ตั้งเป้าคะแนน + พยากรณ์ + บอกพอ/ขาด | สตาฟ/ผู้บริหาร |

---

## 4. การใช้งานจริง — เดินงานภาคสนาม

### ลูกทีมเก็บฐานเสียง 1 คน (Part 1)
```
เลือกชาวบ้าน → เก็บข้อมูลบัตรประชาชน (3 วิธีตามอุปกรณ์):
   • เสียบชิป (แม่น 100%)  • ถ่ายรูป+กรอกเบอร์+กดส่ง (คนสูงอายุ)  • พิมพ์เอง
      ↓
เทียบ "บัญชีผู้มีสิทธิ์" อัตโนมัติ:
   [มีสิทธิ์เขตนี้] → ดึงชื่อทางการ (สะกดถูก 100%)
   [อยู่คนละหน่วย] → เก็บเข้าบัญชีพิเศษ (ไม่เสียเสียง)
   [ไม่มีสิทธิ์]   → เตือน ไม่เก็บ (ไม่เปลืองเวลา)
      ↓
ยืนยัน 2 ชั้น:  (1) ตัวตนจริง (ชิป/เซลฟี่/GPS)   (2) เจตนาจริง (เจ้าตัวกดยืนยันผ่านมือถือตัวเอง)
```
> จุดเด่น: ยืนยัน "เจตนา" แยกจาก "ตัวตน" → จับเคส "เอาบัตรคนอื่นมาแอบอ้าง" ได้
> และกันลูกทีมปั่นยอดด้วยเลขบัตรปลอม

### หัวหน้าทีม/สตาฟ ตรวจยอด (Part 2)
- เห็น **กรวยยอด**: เก็บ 50 → ผ่านเกณฑ์ 46 → ยืนยันแล้ว 41 → ใช้งานได้จริง 41
- **กระทบยอดสองฝั่ง**: ลูกทีมบอก 50 ระบบเห็นไม่ครบ → ระบบบอกชัดว่า "หายไปไหน เพราะอะไร"
- จัดการรายการ: ซ้ำ / นอกเขต / รอยืนยัน · ปิดยอดรายงวด (เซ็นรับสองฝั่ง)

### ผู้บริหารวางกลยุทธ์ (Part 3)
- ตั้งเป้า เช่น "อยากชนะด้วยความมั่นใจ 80%"
- ระบบพยากรณ์ (จำลอง 2,000 รอบ จากผลจริงครั้งก่อน 185 หน่วย) → บอกโอกาสชนะแต่ละเขต + ฐานเสียงพอ/ขาด/เกิน
- ย้ายกำลังจากเขตที่เกิน ไปเสริมเขตที่เสี่ยง

## 5. คุณค่าที่ได้
| ปัญหาเดิม | WinVote แก้ให้ |
|---|---|
| ฐานเสียงปลอม/นับซ้ำ | เลขบัตร + เทียบบัญชี + ยืนยันเจตนา → ของจริงเท่านั้น |
| ชื่อสะกดผิด | ดึงชื่อทางการจากบัญชี → ถูก 100% |
| ไม่รู้ว่าพอชนะไหม | โมเดลพยากรณ์ → รู้ว่าต้องเพิ่มอีกกี่เสียง |
| ข้อมูลขัดกันสองฝั่ง | กระทบยอด → ตรงกันเสมอ ตรวจย้อนได้ |
| เน็ตไม่ดี/คนสูงอายุ | ออฟไลน์ + โหมดง่าย → ใช้ได้จริงในสนาม |

## 6. สถานะปัจจุบัน
- **ดีไซน์ครบทั้งระบบ** (พิมพ์เขียวระดับ A−) · โค้ดแกน build ผ่าน · มีข้อมูลจริง 185 หน่วย
- **กลไกสำคัญบางส่วนยังออกแบบไว้ กำลังทยอย build** (คะแนนตรวจรวม ~54/100 — ดู Expert Audit)
- **ตัวขวาง:** ฐานข้อมูล WinVote รอเปิด (ติดลิมิต Supabase free 2 โปรเจกต์)
